Frequently asked questions

We send a live HTTP request to the URL you enter from our servers. If the server responds with a valid HTTP status code, the site is up. If the connection times out or returns a server error, the site is down. This is done in real time — no cached results.

When we check from our servers and the site is reachable, but you can't access it, the issue is likely local — try clearing your browser cache, flushing DNS, or switching networks. If our check also fails, the site is globally down for everyone.

HTTP 200 means the page loaded successfully. 301/302 is a redirect (site is up). 403 means access is forbidden (but the server is running). 404 is not found. 500–503 indicates a server error — the site is likely down. Timeouts mean no response at all.

High response times usually mean the server is overloaded, there's a network bottleneck, or the website is located far from our checking server. Times above 2,000ms typically indicate performance issues even if the site technically responds.
